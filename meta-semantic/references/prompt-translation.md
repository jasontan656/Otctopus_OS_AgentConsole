# Prompt 翻译协议

## 1. 目标
- 把用户自然语言翻译成可执行、可检查、低歧义的语义描述。

## 2. 标准流程
1. 抽取动作词。
2. 抽取范围词。
3. 抽取保留项与删除项。
4. 抽取禁止项。
5. 用 [semantic-canon.md](semantic-canon.md) 覆盖模型默认解释。
6. 输出一版简洁的“执行语义”。

## 3. 输出格式
- 动作：要做什么。
- 范围：对哪些对象生效。
- 保留项：什么必须保留。
- 删除项：什么必须清掉。
- 禁止项：什么绝对不要做。
- 推断：只有在确实存在推断时才写。

## 4. 示例

### 示例 A
- 用户原话：`把这块清理掉`
- 执行语义：
  - 动作：删除该块及其直接死引用。
  - 范围：当前被指向的代码或文档片段。
  - 删除项：目标对象、空壳包装、无用兼容残留。
  - 禁止项：不要备份、不要注释保留、不要新增兼容层。

### 示例 B
- 用户原话：`先对齐这套说法，再继续写`
- 执行语义：
  - 动作：先统一术语和语义，再继续后续写作或实现。
  - 范围：当前任务涉及的提示词、文档、命名或接口描述。
  - 禁止项：不要在未定义参照系的情况下直接改写。

## 5. 升级条件
- 若原句无法判断动作对象是谁，才追问对象。
- 若原句的禁止项会直接改变执行结果，才追问禁止项。
- 若只是实现细节存在多种等价方案，不要把实现选择误报成语义歧义。
