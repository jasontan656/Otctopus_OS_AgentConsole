# 语义总表

## 1. 使用方式
- 当用户表达与模型默认理解可能偏离时，优先查本表。
- 每个条目都按四段解释：
  - 原始表达
  - 归一化意图
  - 必须包含
  - 明确排除

## 2. 高优先级条目

### 2.1 清理掉
- 原始表达：`清理掉`
- 归一化意图：删除目标对象，并消除继续保留它的可见痕迹。
- 必须包含：
  - 直接删除目标内容
  - 同步删除依赖它而存在的空壳、兼容残留或无用引用
- 明确排除：
  - 先备份再删
  - 注释掉替代删除
  - 仅取消引用但保留主体
  - 保留“以后可能还有用”的兼容层

### 2.2 删掉
- 原始表达：`删掉`
- 归一化意图：从目标范围中移除指定对象。
- 必须包含：
  - 删除指定对象本身
  - 清理由此产生的直接死引用
- 明确排除：
  - 自动备份
  - 自动改名保留
  - 注释保留

### 2.3 不留痕
- 原始表达：`不留痕`
- 归一化意图：不要留下为了“保守”而额外保留的替代痕迹。
- 必须包含：
  - 不额外创建 fallback、alias、compat layer
  - 不保留注释化尸体
- 明确排除：
  - 备份文件
  - 临时兼容代码
  - 占位实现

### 2.4 对齐
- 原始表达：`对齐`
- 归一化意图：让目标对象与参照对象在结构、语义、行为或格式上保持一致。
- 必须包含：
  - 先明确参照系
  - 明确需要对齐的是语义、结构、接口还是文风
- 明确排除：
  - 只做表面重命名
  - 没有参照就自称“已对齐”

### 2.5 收敛
- 原始表达：`收敛`
- 归一化意图：减少分叉、减少同义漂移、减少重复表达，统一到少数稳定规则。
- 必须包含：
  - 明确保留哪一个标准写法
  - 清理冗余或冲突写法
- 明确排除：
  - 同时保留多套近义规则
  - 以“兼容”名义继续扩散别名

### 2.6 保留
- 原始表达：`保留`
- 归一化意图：显式留下对象，并在后续动作中避免误删。
- 必须包含：
  - 明确保留对象与范围
- 明确排除：
  - 推断式保留一切相关对象
  - 因为出现“保留”就停止所有清理动作

## 3. 解释优先级
1. 本轮 prompt 中更具体的定义，优先于本表的通用定义。
2. 本表定义，优先于模型默认惯性解释。
3. 若两个条目都命中，优先采用限制更强、禁止项更明确的解释。

## 4. 扩充规则
- 新增条目时，优先补高频误解词。
- 每个条目都要写“明确排除”，否则容易被模型回退到默认常识。
- 如果一个词在不同上下文有不同含义，不要强行合并，拆成多个上下文条目。
