# Dual Purpose Input Contract

- 任何用户描述都被解释为：
  - 产物侧改写信号
  - 技能侧演化信号
- 产物侧信号至少驱动 `mother_doc` 架构、代码地图、使用说明与实验性协议样例刷新。
- 技能侧信号至少驱动规则说明、生成逻辑、lint 规则、tmux subagent 主链与治理流程固化。
- 双重目的输入必须先走 factory，再走 `$Meta-Enhance-Prompt`；禁止把双重目的输入直接回填写盘。
