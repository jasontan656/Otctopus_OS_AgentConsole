# Lint Rulebook

- 规则 1：必备目录与必备文档必须存在。
- 规则 2：Markdown 文档必须具备完整 frontmatter，且 frontmatter 不得只剩 `doc_id / doc_type / topic` 三字段语义空壳。
- 规则 3：禁止出现“文档知识库”。
- 规则 4：root index 与 manifest 必须引用同一套受管文档。
- 规则 5：upstream collaboration 与双重目的输入合同必须显式存在。
- 规则 6：运行合同必须显式承载 `$Meta-Enhance-Prompt`、tmux subagent、`$Functional-Analysis-Runtask` 九阶段与 keyword-first edit 决策。
- 规则 7：实验产物必须显式承载文档关系、消费边界、writeback 语义与下游锚点。
- 规则 8：若 background subagent 没有 implementation / validation 证据，则整个闭环视为失败。
