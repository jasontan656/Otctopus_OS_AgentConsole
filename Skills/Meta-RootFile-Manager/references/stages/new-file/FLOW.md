# New-File Workflow

1. Define the new governed filename or file type.
2. Add the matching structure template.
3. Update scan rules, lint coverage, and supporting tool behavior.
4. Validate dry-run behavior against the user's expected governance shape.
5. Sync mirror and installed skill assets, then complete repo traceability.
