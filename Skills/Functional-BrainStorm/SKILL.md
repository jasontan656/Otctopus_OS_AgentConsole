# Functional-BrainStorm

## 1. 模型立刻需要知道的事情
### 1. 总览
- 本技能用于外部调研、实现思路发散、方案对比、风险与边界挖掘，以及最终落地路径收敛。
- 本技能是纯方法论技能；不提供本地 CLI，不负责写入文件，不负责安装、同步或治理目标技能结构。
- 本技能的价值是把“看外部资料”变成“提炼模式、解释原因、收敛建议”。

### 2. 技能约束
- 优先看一手源与高信噪比资料：
  - 官方文档
  - 官方博客 / 工程博客
  - GitHub 仓库与 issue / discussion
  - 公开架构分享
  - 产品帮助中心 / 定价页 / 发布日志
- 不要只堆外部资料；必须区分：
  - 证据
  - 推断
  - 建议
- 不要只说“别人怎么做”；要说明：
  - 为什么这样做
  - 适用前提
  - 成本与边界
- 如果信息新鲜度可能影响结论，必须先查最新资料再给建议。
- 外部调研的目标不是抄方案，而是抽出可迁移模式并收敛成当前任务可执行的方向。

### 3. 顶层常驻合同
- 优先输出“模式、权衡、适用条件、落地建议”，而不是长篇资料摘抄。
- 当外部案例彼此冲突时，必须显式指出冲突点，而不是强行拼成一个结论。
- 当结论依赖假设时，必须把假设说清楚。
- 如果用户最终要落到当前 repo 或当前技能体系，结尾必须给出可接续的下一步建议。

## 2. 技能正文
### 1. 功能入口
- [大厂经验调研]
  - 作用：调研成熟产品、平台或大厂团队在相近问题上的真实做法。
  - 优先站点：官方文档、工程博客、架构分享、发布日志、GitHub 官方仓库。
  - 输出：已观察模式、共同做法、差异点、适用前提、可借鉴部分。

- [实现思路发散]
  - 作用：围绕一个问题发散多种可能实现路线，而不是过早收缩到单解。
  - 优先站点：官方文档、示例仓库、issue/discussion、技术博客、方案对比文章。
  - 输出：至少 3 条候选路线，每条都有优点、缺点、依赖前提与失败风险。

- [方案对比收敛]
  - 作用：对多个候选路线做取舍，收敛出当前阶段最合适的方案。
  - 优先输入：已有候选方案、外部证据、当前项目约束。
  - 输出：推荐方案、放弃方案、放弃原因、最终选择依据。

- [风险与边界挖掘]
  - 作用：提前暴露实现中的隐性风险、边界条件、迁移成本与维护成本。
  - 优先站点：issue/discussion、故障复盘、已知限制文档、兼容性说明。
  - 输出：关键风险列表、触发条件、缓解思路、必须先确认的问题。

- [落地路径建议]
  - 作用：把前面的调研与脑暴结果收敛成一条当前可执行的落地路径。
  - 优先输入：证据、候选路线、项目现状、边界条件。
  - 输出：建议顺序、最小起步方案、验证点、后续扩展方向。

### 2. 输出要求
- 默认输出顺序：
  - 问题重述
  - 外部证据
  - 模式归纳
  - 方案建议
  - 风险与下一步
- 当用户只要求发散时，可以减少收敛强度；但仍要保留每条候选方向的适用前提与边界。
- 当用户要求最终建议时，必须明确推荐项与不推荐项。
