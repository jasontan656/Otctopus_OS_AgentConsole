# mother_doc 阶段合同

## 当前动作要完成什么
- 把 live requirement 写入当前 mother doc 体系，而不是新造一棵混写大树。
- 让主题沿既有层级从人类叙事逐步过渡到机械叙事。
- 当更深锚点主题已由用户明确命名时，再继续向下落盘，并同步整体架构文档。

## 当前动作必须满足什么
- 顶层先维护 index、product baseline、common rules、entry map 一类常驻文档，不把 implementation 语义塞回这些节点。
- 任何新的更深锚点主题都必须由用户显式指定。
- 真实写回后必须完成 lint、root index refresh 与 client mirror sync。
