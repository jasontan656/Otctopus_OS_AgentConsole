# Meta Keyword First Edit（删改写优先）

## 概览
对所有修改任务应用固定优先级：
1. 删掉重写
2. keyword first 替换
3. 新增

核心主张：
- 不要把某个局部目标当成唯一目标，然后对片段单独打补丁。
- 先理解完整对象的架构、开发意图、消费者和上下文，再决定如何改。
- 当现有表达、结构或逻辑已经偏了，最高优先级不是“最小改”，而是删掉后按正确意图整体重写。

## Read-First Gate（逐字通读）
在提出改动前，必须先完整阅读相关对象，最小单位也必须是能保持语义闭环的完整单元：
- 文档：完整章节或完整文档
- 代码：完整函数、完整类、完整模块，必要时向上读到调用链
- 配置/工作流：完整配置块、完整任务流

禁止行为：
- 只读到某一小段就直接改那一小段
- 只围绕单一报错或单一目标做局部修补
- 在未确认整体意图前先加“兼容层”“legacy 标记”“禁止式约束句”

## 优先级一：删掉重写（最高优先级）
适用条件：
- 现有结构已经偏离真实意图
- 局部补丁会造成上下文断裂
- 为了满足一个目标而容易误伤底层逻辑、相邻逻辑或整体表达
- 文案/规则/代码虽然“能修”，但修完大概率变丑、变绕、变补丁化

执行原则：
- 先理解原对象想表达什么、服务谁、为什么这样设计
- 删除失效结构后，按当前正确意图整体重写
- 重写后结果必须是连贯的、完整的、优雅的，而不是补丁堆叠

删除纪律（硬约束）：
- 如果某段逻辑、某条规则、某段文案已经不需要，直接删除
- 不要保留为 `legacy`
- 不要用“兼容旧逻辑”当默认动作
- 不要删除后再补一句“禁止再做某事”“你必须不……”来当补丁
- 真正不需要的东西，删除本身就是最干净的表达

删除确认门（硬约束）：
- 只要本次任务存在删除或潜在删除意图，就不能直接动手删除
- 必须先显式列出“将被删除/被覆盖/被整段替换”的具体对象
- 对象必须尽量具体到：文件、章节、段落、规则块、函数、类、逻辑分支、脚本说明段
- 列完后必须请求用户确认，确认前不得执行删除
- 不得把“用户大致说了要瘦身/简化/重写/移除偏离内容”视为对全部删除范围的自动授权
- 只有当删除范围已经被用户明确确认后，才允许继续执行 rewrite/delete

## 优先级二：keyword first 替换
只有在“无需整体重写”时，才进入替换层。

核心主张：
- 语言是一种高压缩载体，很多时候换一个字、一个短语、一个判断条件，就足以改变整句或整段的语义。
- 因此，在无需删掉重写时，应优先寻找最小但高质量的关键词/语义替换，而不是新增补丁句。

执行方式：
1. 识别目标意图与现有偏差
2. 定位最小触发集合：词、短语、条件、命名、判断节点
3. 给出替换对：`old -> new -> reason`
4. 选择最小替换集合，完成语义纠偏

适用边界：
- 替换后整体结构仍然成立
- 替换后不会留下历史残影或语义冲突
- 替换足以达成目标，无需新增解释层

## 优先级三：新增（最后手段）
只有在以下情况才允许新增：
- 删掉重写不合适
- keyword 替换不足以满足目标
- 新信息在现有结构中确实不存在，且不可由删除/重写/替换获得

新增要求：
- 必须证明“为何重写不足、为何替换不足”
- 新增内容必须是最小必要增量
- 新增后不得制造重复语义、兼容壳、补丁味规则

## 核心流程
1. 先完整阅读目标对象，确认架构与开发意图
2. 判断是否应进入“删掉重写”
3. 若无需重写，再判断是否可由 keyword 替换完成
4. 仅在前两者都不足时，才允许新增
5. 若存在无用逻辑、无用规则、无用文案，显式删除，不保留 legacy/兼容残留

## 调试与改写场景用法
1. 先回答“这个对象本来应该表达什么/做什么”
2. 再回答“当前实现为什么偏了”
3. 若偏差是结构性的，删掉后整体重写
4. 若偏差只是语义性的，做 keyword first 替换
5. 若目标是全新信息，最后再新增

## 输出契约
- 必须先给出改写决策：
  - `rewrite`
  - `replace`
  - `add`
- 若选择 `rewrite`：
  - 必须说明为何局部修改会导致断裂或误伤
  - 必须说明哪些旧内容会被直接删除
  - 若删除范围尚未被用户确认，先列删除清单并请求确认，不能直接改
- 若选择 `replace`：
  - 必须给出替换清单：`old -> new -> reason`
- 若选择 `add`：
  - 必须说明为何重写与替换都不足

## 语言规则
- 对话输出必须使用**中文**。
- 起草文档时默认使用**中文**，除非用户另有指定。
- 代码块、代码注释和代码文件必须使用**English**；不要在代码中引入中文。
