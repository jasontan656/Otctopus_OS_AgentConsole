# OPERATING_MODEL_BASELINE

- `Octopus_OS` is the single project operating surface for development, deployment, and future operations.
- Humans issue intent, read outcomes, and answer unresolved questions.
- The Octopus AI writes Mother_Doc, implements code, runs tests, brings services up, and closes evidence.
- `Octopus_OS` is one project with many deployable container roots; do not treat container boundaries as separate products unless the user explicitly redefines the project boundary.
- `Mother_Doc` is the authoritative documentation-and-graph container for the whole project, not a side folder.
